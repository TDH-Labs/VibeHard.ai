WHY A HUMAN REVIEW DOESN'T COST EXTRA

Every review stays narrow by design.

When a build hits something a gate can't resolve on its own, it doesn't sit in a queue waiting for an expensive open-ended engagement. The gate has already localized the exact flagged slice — the one policy, the one field, the one migration — so a reviewer isn't auditing your whole app. They're looking at one specific thing, with the context needed to judge it and nothing more.

That's the difference between a 15-minute specialist visit and a full workup, and it's why this can be baseline behavior instead of a premium add-on. Every tier gets it. There's no "upgrade to unlock a real person" — a held build routes to review the same way whether you're paying $0 or $99 a month. The tiers below set your daily build throughput. They don't set your safety margin.

QUESTIONS THAT ACTUALLY COME UP

Before you pick a plan.

Do I pay more if my build gets flagged for review?

No. Routing a held build to a reviewer is baseline gate behavior for anything that needs judgment instead of a mechanical fix, and it's included at every tier equally.

What happens when I hit my daily build limit?

Builds already running finish normally. New builds wait until the next day's allotment, or you move to a tier with more throughput. Nothing you've shipped gets touched.

Can I use my own model provider?

Pro supports bringing your own model key as an advanced option, if you'd rather run builds against your own account instead of the platform's.

Is the database actually mine?

Every build gets its own dedicated database project — never a shared table commingled with any other customer's data. VibeHard provisions and manages it by default, so there's no separate account for you to set up.

What if a build never converges?

The repair loop is bounded. If it exhausts its rounds without a clean pass, the build is held and routed to a person with every finding attached — it never ships broken, and it never just sits there either.
