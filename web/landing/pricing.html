The part that never changes.

Twelve gates

Every build runs the same fixed line: code scanned for exploitable patterns and committed secrets, dependencies checked against known vulnerabilities, migrations executed on a real database, tenant isolation attacked live, acceptance criteria run as tests, every promised feature checked off in code.

The Sentinel

When all twelve gates pass, the Sentinel signs the build. That signature is the only door to deploy, and no plan buys a way around it.

Findings you can read

A build that can't pass is held, with each finding in plain English. Rerun the fix loop or refine the request and build again — a hold never costs extra.

A live deploy

A passing build ships to a live URL you can put in front of clients, each project on its own database.

Questions

Before you pick.

Why is this more than a $20 chatbot?

A chat subscription sells replies. A gated build runs a full pipeline — spec, code, then the twelve-gate inspection with a real database behind it — and a live project stays hosted after it ships. Builds and live projects are what cost money to provide, and they're the only things a plan meters.

Do I pay more when a build is held?

No. A hold is the line working, and it works the same on every plan. You see the findings, rerun the fix loop or refine the request, and build again. A held build never generates a charge.

What happens when I hit my daily limit?

Builds already running finish normally. New builds wait for the next day's quota, or you move to a plan with more. Nothing you've shipped is touched.

Is the database actually mine?

Each project gets its own dedicated database — never a table shared with another customer. VibeHard provisions and manages it, so there's no separate account to set up.
