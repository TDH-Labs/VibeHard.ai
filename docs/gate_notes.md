- **CDN blind spot closed (2026-06-21):** RLS source scanning now includes server-rendered templates (.html/.htm/.ejs/.hbs/.handlebars/.pug), and Supabase is detected via multiple signals — npm import, CDN `window.supabase` global, a `*.supabase.co` URL, and `SUPABASE_*` env names — so a client-side/CDN Supabase app can no longer pass the gate vacuously when it ships no RLS migration.
- **Tenant-scoping RLS warning (2026-06-21):** Added `rls-policy-authenticated` — a MEDIUM, non-blocking WARN for tables whose RLS `using` clause authorizes any authenticated user (`auth.uid() is not null` / `auth.role() = 'authenticated'`) instead of scoping rows to their owner. Not the CVE pattern (anonymous is still blocked), so it's surfaced for a reviewer rather than blocked — legitimate for single-tenant, a cross-tenant leak otherwise. Seen in 3 of 4 dogfood Supabase apps (team board, feedback, clinic); correctly silent on the per-user notes app and the non-Supabase control. The CLI now prints non-blocking warnings on the PASS path too.
